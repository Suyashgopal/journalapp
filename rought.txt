STEPS TO INTEGRATE MONGODB WITH A SPRING BOOT PROJECT

STEP 1: ADD MONGODB DEPENDENCY

Add the following dependency to your pom.xml file and reload Maven:

<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-data-mongodb</artifactId>
</dependency>

This dependency enables Spring Data MongoDB support in your Spring Boot application.


STEP 2: CONFIGURE MONGODB IN application.properties

OPTION A: STORE DATA LOCALLY (LOCAL MONGODB)

spring.data.mongodb.host=localhost
spring.data.mongodb.port=27017
spring.data.mongodb.database=journaldb
spring.data.mongodb.auto-index-creation=true

auto-index-creation=true automatically creates indexes which helps with references and performance.


OPTION B: STORE DATA ON MONGODB ATLAS (CLOUD)

1. Create a cluster and database in MongoDB Atlas
2. Create a database user (username and password)
3. Copy the MongoDB connection URI

Add the following to application.properties:

spring.data.mongodb.database=journaldb
spring.data.mongodb.uri=mongodb+srv://<username>:<password>@cluster0.z9pqr6d.mongodb.net/journaldb?retryWrites=true&w=majority
spring.data.mongodb.auto-index-creation=true

NOTE: Do not hardcode credentials in real projects. Use environment variables.


STEP 3: USE @Document ANNOTATION

Add @Document on the class you want to map to MongoDB.

Example:
@Document(collection = "journal_entries")
public class JournalEntry {
}

@Document marks a Java class as a MongoDB document and maps it to a MongoDB collection.


STEP 4: CREATE REPOSITORY INTERFACE

For each @Document class, create one repository interface.

Syntax:
public interface JournalEntryRepository extends MongoRepository<JournalEntry, ObjectId> {
}

MongoRepository provides built-in methods such as:
save()
findById()
findAll()
deleteById()


STEP 5: CREATE SERVICE CLASS

Create a service class and annotate it as a Spring component.

Example:
@Service
public class JournalEntryService {

    @Autowired
    private JournalEntryRepository repository;

    public JournalEntry save(JournalEntry entry) {
        return repository.save(entry);
    }

    public void deleteById(ObjectId id) {
        repository.deleteById(id);
    }
}

The service layer contains business logic and keeps the application clean and maintainable.


FLOW SUMMARY

Entity (@Document)
Repository (MongoRepository)
Service (Business Logic)
Controller (REST APIs)

 1xx--informational ((we dont use them

2xx-sucessful---indicate that the request was sucessfully recvd + understood + processed by server
200ok ---the request has been succesfully processed and h server is returning the requested resource the client asked for
201 created ---the request has been fulfilled and a new resource is created as a result
204 no content-- the req was suceessful but there is no respose body (in this operation)  used for oprn like deletion


3xx redirection
these codes indicate that further action is need to complete the request they are used when the client needs to take additional steps to gets its request completed

302 found-- the reqested resource is been temporarey moved to different url when a server sends a 302 code it usually includes a location header feild (of new temp url location) where client is redirected
304 not modified -- the cients cached version of the requested resource is till valid so this code indicates that


4xx client error--indicate that there was an error on the clients part such as malformed request or authentication issue
